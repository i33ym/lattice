How It Works

Three steps to reactive data

Lattice turns your data schema into a live computation graph. Define once, and every insert triggers the right computations in the right order.

01

Define Schema

Declare your table with raw columns and computed columns. Computed columns reference their dependencies and a UDF to produce values.

02

Insert Data

Insert raw data into any column. The engine detects which computed columns are affected and marks them dirty.

03

Auto-Compute

The DAG engine topologically sorts dependencies and invokes UDFs in the correct order. Downstream values materialize automatically.

Core Concepts

Built for reactive pipelines

Every piece of Lattice is designed around one idea: when data changes, everything downstream should update automatically.

DAG Engine

Topological sort ensures computed columns evaluate in dependency order. Cycles are detected at schema definition time, not at runtime.

Computed Columns

Define columns as pure functions of other columns. The engine calls your UDF whenever upstream values change, keeping derived data fresh.

Multimodal Types

First-class support for text, images, audio, video, and vector embeddings. Blobs are stored via pluggable blob stores with lazy loading.

Dirty Tracking

When a cell changes, only its downstream dependents are recomputed. Unchanged branches of the DAG are skipped entirely, minimizing work.

User-Defined Functions

UDFs are gRPC services you write in any language. Lattice invokes them with input cells and writes back the result. Hot-reload supported.

Backfill

Add a new computed column to an existing table and Lattice backfills all historical rows. Progress is tracked and resumable after interruption.

Architecture

Pluggable by design

Lattice separates concerns through clean adapter interfaces. Swap storage, vector backends, and auth layers without changing application code.

Examples

See it in action

From simple schemas to full multimodal pipelines, Lattice keeps your code clean and your data reactive.

Go 
package main

import "github.com/nicholasblaskey/lattice"

func main() {
    store := sqlite.New("data.db")
    engine := lattice.New(lattice.WithStore(store))

    // Define a simple table
    engine.CreateTable(ctx, lattice.TableSchema{
        Name: "users",
        Columns: []lattice.ColumnSpec{
            {Name: "name", Type: lattice.TextType},
            {Name: "email", Type: lattice.TextType},
        },
    })

    // Insert a row
    engine.Insert(ctx, "users", []lattice.Row{
        {"name": "Alice", "email": "alice@example.com"},
    })

    // Query rows
    rows, _ := engine.Query(ctx, "users", lattice.Where("name", "=", "Alice"))
}

// Define a UDF that computes full name from first + last
fullNameUDF := lattice.UDF("grpc://localhost:9090/fullname")
sentimentUDF := lattice.UDF("grpc://localhost:9090/sentiment")

engine.CreateTable(ctx, lattice.TableSchema{
    Name: "reviews",
    Columns: []lattice.ColumnSpec{
        {Name: "author", Type: lattice.TextType},
        {Name: "body", Type: lattice.TextType},
        // Computed: auto-updates when "body" changes
        {Name: "sentiment", Computed: lattice.Computed(sentimentUDF, "body")},
        {Name: "word_count", Computed: lattice.ComputedFunc(func(row lattice.Row) any {
            return len(strings.Fields(row["body"].(string)))
        }, "body")},
    },
})

// Insert - sentiment + word_count compute automatically
engine.Insert(ctx, "reviews", []lattice.Row{
    {"author": "Bob", "body": "Absolutely love this product!"},
})

// Update - downstream columns recompute
engine.Update(ctx, "reviews", lattice.Set("body", "Changed my mind, it's okay."),
    lattice.Where("author", "=", "Bob"))

embedUDF := lattice.UDF("grpc://localhost:9090/embed")
vectorStore := pgvector.New(connStr)

engine := lattice.New(
    lattice.WithStore(pgStore),
    lattice.WithVectorStore(vectorStore),
)

engine.CreateTable(ctx, lattice.TableSchema{
    Name: "articles",
    Columns: []lattice.ColumnSpec{
        {Name: "title", Type: lattice.TextType},
        {Name: "content", Type: lattice.TextType},
        {Name: "embedding", Computed: lattice.Computed(embedUDF, "content")},
    },
})

// Embeddings auto-compute on insert
engine.Insert(ctx, "articles", []lattice.Row{
    {"title": "Intro to Go", "content": "Go is a statically typed..."},
})

// Similarity search powered by pgvector
results, _ := engine.Search(ctx, "articles", lattice.SimilarTo{
    Column: "embedding",
    Query:  "concurrency patterns in Go",
    Limit:  10,
})
Go 
transcribeUDF := lattice.UDF("grpc://localhost:9090/whisper")
summarizeUDF := lattice.UDF("grpc://localhost:9090/summarize")
thumbnailUDF := lattice.UDF("grpc://localhost:9090/thumbnail")

blobStore := minio.New(minioClient, "media-bucket")
engine := lattice.New(
    lattice.WithStore(pgStore),
    lattice.WithBlobStore(blobStore),
)

engine.CreateTable(ctx, lattice.TableSchema{
    Name: "videos",
    Columns: []lattice.ColumnSpec{
        {Name: "video", Type: lattice.VideoType},
        // Chain: video → transcript → summary
        {Name: "transcript", Computed: lattice.Computed(transcribeUDF, "video")},
        {Name: "summary", Computed: lattice.Computed(summarizeUDF, "transcript")},
        // Parallel: video → thumbnail (independent of transcript)
        {Name: "thumbnail", Computed: lattice.Computed(thumbnailUDF, "video")},
    },
})

// Upload video — transcript, summary, and thumbnail compute in parallel where possible
engine.Insert(ctx, "videos", []lattice.Row{{
    "video": lattice.BlobFromFile("./lecture-01.mp4"),
}})
